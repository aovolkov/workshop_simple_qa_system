Ну вот это очень хороший вопрос.

Разумеется, голос и текст - это средства ввода.

Оркестратор же - то, что управляет всеми сервисами и устройствами - это и есть "умная начинка". И он должен быть в самом ассистенте (в общем-то, это и есть ассистент).

В этом плане Алиса - это и есть VINS (серверная система, обрабатывающая входящие голосовые запросы, классифицирующая их по сценариям и интентам, и выделяющая сущности из фраз) и BASS (серверная система, оркестрирующая работу всех сервисов и устройств через их APIs).

В этом плане ассистент ограничен следующим образом:

* возможности внешних систем
* их APIs, доступные для внешнего управления этими системами
* программные интерфейсы окрестратора
* сценарии и интенты ассистента, а также routines

То есть, например:
* есть датчик движения, который может отчитываться о детектированном движении
* есть API у системы, отвечающей за этот датчик, который может либо отвечать на вопрос, есть ли движение, либо дёргать внешний API (ручку, как говорят в Яндексе) о произошедшем событии
* есть оркестратор, умеющий только опрашивать раз в N моментов, есть ли движение
* нет поддержки routine, которая бы запускалась в случае, если датчиком определено движение

И все, мы не можем воспользоваться этим датчиком
